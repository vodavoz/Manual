# Регламентные задание Телефония

## Мессенджер: Синхронизация сообщений

Включает/отключает обмен сообщениями между системой взаимодействия 1С и сервисом whatsapp. Входящие сообщения поступают моментально, время отправки исходящих сообщений будет зависеть от установленного расписания регламентного задания.

Стандартное расписание: 

каждый день; с 8:00:00 по 21:00:00 каждые 15 секунд.

Рекомендуется устанавливать интервал не более 60 с.

## Телефония: Загрузка истории звонков

По частям загружает историю телефонных звонков с сервера телефонии. При запуске проверяет идентификатор последнего загруженного звонка и выполняет загрузку порции истории звонков в базу 1С.

Стандартное расписание: 

каждый день; с 8:00:00 по 21:00:00 каждые 120 секунд.

Если регламентное задание отключить журнал звонков в подсистеме телефонии заполняться не будет.

## Телефония: Обмен с сервером телефонии контактными данными

Заполняет информацию о владельцах номеров телефонов (компаниях и контактных лицах) в записях журнала звонков, загруженных за последние 3 месяца, если они еще не были заполнены ранее.

Стандартное расписание: 

один день, по дням недели (суббота); с 20:00:00 один раз в день.

После обновления контактной информации в карточке контрагента ищет документы телефонный звонок с аналогичным номером телефона и устанавливает связь между документом звонок и партенром/контрагентом/контактом.
Регламентное задание обрабатывает звонки в которых не проставлен контрагент за последние 3 месяца. Это может оказывать серьезную нагрузку на сервер, поэтому рекомендуется запускать в нерабочее время, например раз в неделю по субботам.

## Телефония: Обмен с сервером телефонии событиями

Устанавливает соединение с сервером телефонии для получения данных о выполняемых звонках. Фоновое задание может мешать обновлению информационной базы. Настраивайте расписание с учетом окна для проведения технических работ.

Стандартное расписание: 

каждый день; с 8:00:00 по 21:00:00 каждую секунду.

Доступно, если канал передачи событий установлен в значение long-polling соединение. Отвечает за открытие карточки клиента в момент входящего звонка, если используется longpoll соединение от сервера 1С к АТС.

## Телефония: Обновление контактных данных (оперативно)

Заполняет информацию о владельцах номеров телефонов за последние 5 дней.

Стандартное расписание: 

каждый день; с 8:00:00 по 21:00:00 каждые 900 секунд.

Делает то же самое, что и обновлять контактные данные только для звонков за последние 5 дней.

## Телефония: Обновление таблицы маршрутизации

Заполняет таблицу маршрутизации для голосового меню данными о клиентах и сотрудниках. Ранжирует сотрудников по степени их участия в работе с клиентом и автоматически выявляет ответственных.

Стандартное расписание: 

каждый день; с 8:00:00 по 21:00:00 каждые 900 секунд.

Регламентное задание обрабатывает события в базе данных, настроенные в разделе отслеживаемые взаимодействия, строит таблицу маршрутизации которая используется, как для модуля умной маршрутизации, когда клиенту генерируется персональное голосовое меню, так и для функции перехвата на ответственного, если в настройках подсистемы указан этот вариант перехвата.
Если выключить это задание, то таблица маршрутизации станет неактуальной и вышеописанные функции будут работать с устаревшими данными.

## Телефония: Отправка метрики разработчикам

Оно есть в списке регламентных и фоновых заданий раздела "Администрирование", в настройках подсистемы телефонии его нет.

Стандартное расписание: 

каждый 7-й день, по дням недели (воскресенье); с 20:00:00 один раз в день.

Раз в сутки отправляет данные по используемым конфигурации 1С; подсистеме телефонии; версии подсистемы телефонии; версии АТС; диалплана; статистике использования объектов подсистемы.
